Data Types in JS

Javascript hase the following Data types.

These are further divided into 2 types: primitive and referential, based on how they are treated in memory

Primitive data storage

Primitive data types are stored in Stack memory (and hence they are quick to be assessed). There is less space available though. When we store a primitive into the memory it stores value into the stack and locks the name to it as a accessor.

Reference data type Storage

Reference data types are stored in heap by contrast and a pointer to it to the stack. The variable name is locked onto the pointer for access

Assignment Differences

When we make a copy of the primitive data type we make a copy of the value and are stored separately. Which means the values of the different variables can be manipulated separately.

When we make a copy of the reference data type we only copy the pointer to the new const. Which means once the value of the object in heap is overwritten all the variables attached to the pointer get updated.
